Progressive Web App (PWA)

Characteristics

Usage of Service Workers
The use of a service worker allows PWAs to work offline or on low-quality networks. Service workers intercept network requests, cache, and update the cache. It checks for updates in the background and refreshes content so that the user sees the newest version. The service worker also helps the PWA send out push notifications when the web app is not open. Like traditional native apps, users can be re-engaged back to the app through promotions, updates, and important events.

App-like, discoverable, and installable
PWAs mimic the navigation and interface of native apps. They are responsive, working well with phones, tablets, and desktops. PWAs are accessible through URLs and they have a manifest file for web apps that search engines recognize as a web application. Moreover, the web app manifest allows users to add the PWA to their home screen without going through an app store. The manifest file specifies metadata for the installation like name, icon, start URL, and display properties, providing a native-like experience.
